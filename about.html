About visualize-pbft

visualize-pbft is an ECS 235A final project that facilitates understanding the robustness of Practical Byzantine Fault Tolerance (pBFT), a distributed systems consensus protocol, by simulating node-to-node communication as queues shared by client and replica threads on backend, and returning message data and results to a web frontend for processing.

The form on the dashboard page allows the user to enter 4 parameters to control the simulation run: number of replicas total, number of Byzantine replicas, number of transactions to execute, and Byzantine behavior (one behavior applies to all chosen Byzantine replicas). To pass form option validation, ensure that your inputs are reasonable, satisfying the following requirements:
  • all textbox inputs are valid integers.
  • the number of replicas is between 2 and 8 inclusive.
  • the number of Byzantine replicas is between 0 and 8 inclusive.
  • the number of transactions is between 1 and 10 inclusive.
  • if the number of Byzantine replicas is set to a nonzero number and "None" is specified for Byzantine behavior, the input is invalid.
  • the number of Byzantine replicas must be strictly less than the number of replicas total.
